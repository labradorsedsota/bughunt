## 经验记录

### git push 代理问题 (2026-04-18)
- git push 被 SIGKILL 多次：因为代理环境变量导致连接超时
- 解决方案：`env -u http_proxy -u https_proxy -u HTTP_PROXY -u HTTPS_PROXY -u all_proxy -u ALL_PROXY no_proxy="*" NO_PROXY="*" git push origin main`
- 比之前只用 `no_proxy="*"` 更彻底，需要同时 `-u` 清除所有代理变量

### mint-ui 项目特点 (2026-04-18)
- 基于 cooking + webpack 构建，首次启动慢（~10s），切 commit 后重启快
- yarn.lock 切换 commit 时需要先 `git checkout -- yarn.lock`
- mano-cua 平均跑 50-74 步，接近 15min 硬超时
- 多个 commit 之间可复用 node_modules，只需 `npm install --legacy-peer-deps`
